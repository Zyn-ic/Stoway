InventoryData

Core data structure for player inventory state

This class is instantiated per-player and holds all inventory state including items, slots, and equipped status. It uses the ItemsByID lookup table for O(1) stack searches.

Type Definition Luau 
export type ItemInstance = {
    UUID: string,
    Id: string,
    Amount: number,
    Metadata: { [string]: any }?
}

export type InventoryData = {
    Player: Player,
    Items: { [string]: ItemInstance },      -- UUID -> Item
    ItemsByID: { [string]: {string} },    -- ItemId -> UUIDs (O(1) lookup)
    Hotbar: { [number]: string? },         -- Slot -> UUID
    Storage: { [number]: string? },        -- Slot -> UUID
    EquippedSlot: number?,
}

AddItem(itemId, amount, metadata?)

Adds items with auto-stacking O(1) Lookup

First attempts to stack with existing items using the ItemsByID lookup table. If stacking fails or isn't possible, finds an empty slot (Hotbar first, then Storage).

Flow Pseudocode 
1. Check if stacking enabled
2. Look up existing UUIDs via ItemsByID[itemId]  ‚Üê O(1)
3. Try to add to existing stacks (respecting MaxStackSize)
4. Create new item instances for remaining amount
5. Find empty slots (Hotbar priority)
6. Update Items, slots, and ItemsByID

RemoveItem(uuid, amount)

Removes items and maintains lookup table

Decrements stack count or removes item entirely if amount matches. Automatically cleans up ItemsByID entries.
