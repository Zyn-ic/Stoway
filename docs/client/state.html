InventoryStore

Reactive state management powered by Fusion

The client uses a single source of truth in InventoryStore.luau. This module holds the entire inventory state (Items, Hotbar, Storage, Settings) wrapped in Fusion State Objects. UI components observe these objects and update automatically when data changes.

Accessing State Luau 
local Store = require(Client.CoreState.InventoryStore)
local Fusion = require(Packages.Fusion)
local peek = Fusion.peek

-- Get instantaneous state
local state = Store:GetState()
print("Equipped UUID:", state.Equipped)
print("Total Weight:", peek(Store.Weight))

-- Observe changes (in a Fusion scope)
scope:Observer(Store.Weight):onChange(function()
    print("Weight changed to:", peek(Store.Weight))
end)

State Atoms

The core reactive values

Atom Type Description
Items { [uuid]: ItemData } Dictionary of all item data (Name, Amount, Metadata).
Hotbar { [1-9]: uuid } Static slots mapping index to item UUID.
Storage { uuid } Packed array of item UUIDs in storage.
EquippedUUID string? The UUID of the currently equipped item.
Weight number Current total weight of the inventory.
Settings table Client copy of player-specific settings (Limit, etc.). Controls conditional UI rendering (e.g., hiding weight labels or stack counts).

Helper Methods

Convenience functions for data access

Method Returns Description
GetState() table Returns a snapshot of the current state (unwrapped).
GetUUidFromIndex(index, type) string? Safe lookup for UUID given a slot index and type.
DoesItemExist(uuid) ItemData? Returns item data if it exists, nil otherwise.
